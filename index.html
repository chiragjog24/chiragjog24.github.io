Multi-Agent Pipeline Orchestrator
/ship-feature
Takes a requirement or Linear ticket and drives it through the full development lifecycle: PO triage, conditional product review, UX design, architecture planning, parallel development in git worktrees, 3-tier testing with classified fix loops, code review, UAT on Railway preview, and squash merge to main.

4e
Post Inline PR Comments
Each finding → inline comment on file:line
Summary table: blocker / warning / suggestion counts

4f · Fix Loop
Max 3 Rounds
Review Fix Loop
Round 0: Full 9-point review
Round 1-2: Scoped re-review (only flagged items)
Dev agent fixes blockers + warnings in worktree
Reviewer verifies: Resolved / Not resolved / Regressed
!
Independent budgets — test fixes (4h) and review fixes (4f) never nest. After test fixes, only a single-pass review runs (no full 4f loop). Prevents 3×3 = 9 cycle explosion.

Error Handling
Agent Failure
Retry / Skip / Cancel
Critical (PM, Arch, Dev, QE, Reviewer): retry or cancel
Non-critical (UX): can skip, proceed text-only
Merge Conflicts
Auto-resolve or Escalate
Dev agent attempts resolution
If failed: present conflicts to user for manual fix
Deploy Failure
Retry / Logs / Skip
Railway preview fails health check
Check logs, retry, or skip (not recommended)
